## Key Patterns

### Pattern 1: Start Simple, Expand If Needed

Don't assume complexity. Start with basic questions, expand if answers reveal more.

### Pattern 2: Recount When Things Change

If user adds unexpected features mid-conversation, stop and recount total scope.

### Pattern 3: Help With Examples

When user says "–Ω–µ –∑–Ω–∞—é", offer common patterns or examples.

### Pattern 4: Confirm Understanding

Regularly summarize what you understood to catch misalignments early.

### Pattern 5: Reality Check

If scope seems unrealistic, help user think through phasing.
